# 无人机可见光与热红外融合：3D 热映射技术路线与工程契约

文档声明：本文仅基于《High-Resolution 3D Thermal Mapping: From Dual-Sensor Calibration to Thermally Enriched Point Clouds》提炼方法主线。具体数据边界、板几何、运行参数、文件字段与目录规则以 [AGENTS.md](AGENTS.md) 以及 [docs/](docs/) 下对应文档为准，本文不重复这些事实表。

## 0. 论文主线概述

- 应用背景：高分辨率 3D 热模型广泛应用于结构健康监测、文化遗产文档化以及精准农业等领域，通过将亮度温度数据融合到 3D 几何特征中来检测异常和评估材料劣化（如隐蔽裂缝、湿气侵入或隔热损耗）。
- 硬件平台：基于双传感器（RGB + thermal）采集系统。
- 核心挑战：热红外传感器空间分辨率过低（通常在 $160 \times 120$ 到 $640 \times 512$ 像素之间），而可见光相机通常可达 48 MP 或更高。这种巨大的分辨率和视角差异使得建筑细节在热场中不可见，依赖纯热红外影像无法生成稠密精确的 3D 模型。
- 技术主线：
  1. 双相机几何标定与图像去畸变。
  2. 计算热红外与可见光系统之间的交叉光谱单应性或对应关系。
  3. 提取热红外辐射数据并生成绝对温度矩阵。
  4. 使用高分辨率 RGB 影像通过 SfM / MVS 重建三维稠密点云。
  5. 通过射线投射可见性判定、插值与多视图加权融合，将提取的温度值直接映射并富集到三维点云。

## 1. 文献方法事实

### 1.1 采集与元数据

- 论文采用双传感器平台同步采集 RGB 与 thermal 图像对。
- 自动化工作流以 Python 实现，集成计算机视觉库与 Agisoft Metashape API。该方法的优势在于不依赖专有独立平台的逐图手动调整，大幅减少了累积误差。
- 热红外原始辐射数据直接通过厂商 SDK 读取与解析。
- 论文强调输入数据必须可追溯到原始设备元数据和环境参数，以保持辐射计算的精确性。

### 1.2 几何标定与图像去畸变

- 论文使用棋盘风格标定靶，并通过红外灯均匀加热使其在两个光谱域中均表现出高对比度的角点。
- 标定不仅估计双相机的内参，也建立用于后续单应性估计的去畸变基础。
- 针对热图对比度低的问题，论文在标定前处理中引入了自适应对比度增强、强度反转以及针对热红外图像的 Otsu 自动二值化。
- 结果表明，采用该标定策略取得了 81.8% 的棋盘格检测成功率，最终在两个光谱内达到了亚像素级的重投影误差。
- *(具体棋盘格几何、影像尺寸和质量阈值不在本文重复，统一以 [docs/dataset_profile.md](docs/dataset_profile.md) 和 [docs/calibration_model.md](docs/calibration_model.md) 为准。)*

### 1.3 辐射校正与解析

- 传统方法常生成“伪影”或相对温度纹理，而论文的核心目标是将原始热红外量化读数转换为绝对温度矩阵（物理温度值）。
- 该转换过程高度依赖环境温度、相对湿度、材料发射率以及传感器到物体距离等实际环境参数。
- 原始传感器值、中间辐射量和最终温度值必须被分别区分计算与保存，确保辐射校正具有物理意义。
- 输出内容包含明确的计算单位、校正参数和质量标记。

### 1.4 RGB 三维重建 (SfM/MVS)

- 论文以高分辨率且包含丰富高频信息的 RGB 影像作为几何主干，使用 Agisoft Metashape 完成 SfM 与 MVS 重建。
- 重建工作流涵盖照片自动导入、相机对齐与优化、深度图构建和稠密点云生成。
- 最终生成的高精度骨架作为基础载体，承载后续所有的热温度投影。

### 1.5 交叉光谱关联与计算单应性

- 在理想的同平面约束且传感器双绑定的前提下，论文建立了两组影像间的单一平面单应性映射（Planar Homography）。
- 论文通过机器学习与计算机视觉方法系统地比较了 12 种前沿的跨光谱匹配预测算法（包括 DUSt3R, MASt3R, EfficientLoFTR, XoFTR, LiftFeat, RDD, LightGlue, DeDoDe, GIM, RoMa, OmniGlue 和 SuperGlue）。
- 在评估对比中，论文指出 LightGlue 表现极为优异：保留了 99% 的有效匹配，并在 1 px、3 px 和 5 px 阈值下分别达到了 18.2%、65.1% 和 79% 的精确重投影命中率。
- 该稳定的单应性矩阵被用作连接且对齐低宽容度热图至 RGB 对等视角的几何桥梁。

### 1.6 热信息富集与加权分配

- 算法对每个生成的 3D 点执行射线投射遮挡计算，以生成可见性掩膜，判断出该点确切能被哪些热红外图像未遮挡地观测到。
- 依据所求的单应性矩阵，这批点的 RGB 重投影坐标能够被无缝映射到热红外影像坐标系。
- 为维持热图低分辨率下温度值与空间坐标连续性的匹配，对落在非整数坐标的热场进行亚像素获取时采用双线性插值（Bilinear Interpolation）。
- 当一个 3D 点在多个视图中均可见时，为化解视角歧义，论文采取距离和角度混合视角加权融合策略。此基础权重的数学形式为：$w_i = \cos(\theta_i) / d_i^2$，其中 $\theta_i$ 为视点射线与表面法线的夹角，$d_i$ 为相机到点的距离。该加权平均值即被定为终点融合温度。

## 2. 工程契约对齐与约束

- 标定事实与环境配置参数遵循 [docs/runtime_config.md](docs/runtime_config.md) 及全局文件系统架构。
- 标定结果应该完整暴露内参、畸变参数、重投影均方根（RMS）等指标，详细记录依照 [docs/file_formats.md](docs/file_formats.md)。
- 温度矩阵被限制保留契约要求的基本物理环境记录字段。
- 针对重投影的掩膜判定，必须能够准确保存相应点的 `visibility_state` 和 `occlusion_state` 状态。
- 所输出的热富集点云结果应当完全匹配三维结构属性与新添加的温度属性记录，并携带计算置信度（Quality score 与 Support view count 等指标）。
- 若相关匹配计算结果不稳定，应产生降级输出策略或标识异常错误。
