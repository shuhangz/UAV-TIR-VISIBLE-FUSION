# 无人机可见光与热红外影像匹配：TWMM 算法提炼与工程契约

文档声明：本文仅基于《A robust registration method for UAV thermal infrared and visible images taken by dual-cameras》提炼 TWMM (Template Matching with Weights, Multilevel Local Max-pooling, and Max Index Backtracking) 方法的核心学术发现与机制原理。生产字段、路径、阈值与命名规则以 [AGENTS.md](AGENTS.md) 以及 [docs/matching_algorithm.md](docs/matching_algorithm.md)、[docs/file_formats.md](docs/file_formats.md) 为准。

## 0. 文献挑战与主结论

- 匹配挑战（RSTG）：论文指出，可见光与热红外影像之间不仅存在空间分辨率差异，还存在极难克服的辐射差异（Radiation gap）、形状差异（Shape gap）和纹理差异（Texture gap）。热红图常因热扩散导致纹理极为粗糙。
- 传统方法的局限：论文明确表明，由于 RSTG 效应，手工特征基方法（Feature-based）如 SIFT、SURF、ORB，乃至专门针对多模态设计的 RIFT，均难以提取到足够的且重复性高的正确对应点（CPs）。部分区域匹配方法（如 CFOG、HOPC）又因分辨率鸿沟而无法直接应用。
- 核心策略转变：论文放弃特征图主干（如 PC、RIFT、FSC），全面采用基于深度连贯操作的 area-based 方案——即 TWMM 方法。
- 优异性能报告：TWMM 的四个核心步骤，确保了该方法在评估的 600 对涵括不同场景、分辨率和光照的无人机影像上取得了 86.0% 的 correct CP ratio (RCP) 和 96.0% 的 correct matching rate (CMR)。尤其在弱光环境更具极强鲁棒性（RCP 提升 20.7%，CMR 提升 28.1%）。

## 1. 文献方法事实：TWMM 算法四大架构

### 1.1 预处理与尺度初步规范化

- 姿态与平移约束：由于双载荷相机的物理安装特性，TIR 与可见光图像的旋转差异极小。偏移可通过对齐得到一定控制。
- 空间分辨率对齐：为了消除分辨率鸿沟，论文会在匹配前进行尺度规范化。通过系统焦距先验将低分辨率的热红外图像上采样放大。
- 画幅视点对齐：利用中心裁剪（Center-crop）的方式缩小两者边界视点差异。
- *(工程实现中的具体重采样倍率、裁剪窗口和默认值，应由 [docs/runtime_config.md](docs/runtime_config.md) 统一定义与应用。)*

### 1.2 步骤一：原子块的带权模板相似度计算

- TWMM 规避使用过大或过小模板造成的极端缺陷（大模板局部鲁棒好但精度差，小模板精度极高但极易受变形干扰被错误匹配）。
- 系统以 $40 \times 40$ 的 atomic patches 作为第一级最小计算元，认为足够小的原子块内的图像形变可忽略不计（形变低于1像素）。
- 在此基底层输出带权模板的相似度图，为后续上卷操作准备底层精准信号。

### 1.3 步骤二：利用多级特征池化构建高鲁棒金字塔相似度

- 核心突破：TWMM 采用类似于 deep learning 的最大池化技术（Multilevel Local Max-pooling），例如 $3 \times 3$ max-pooling 窗口。
- 这种设计的巧妙在于，它避免了简单均值池化导致的形变失焦，容许了相邻 atomic patches 存在至多 2 像素的局部非刚性变形。
- 通过底向上汇聚，从小响应块成功构建出类似于 $80 \times 80$、$160 \times 160$、$320 \times 320$ 等拥有广阔感受野的、高鲁棒性的大尺度金字塔相似度图。

### 1.4 步骤三：利用最大索引回溯自顶向下提取对应点

- 结合上述金字塔，TWMM 采用自顶向下的 max index backtracking 策略逐层细化确切对应点坐标。
- 匹配从最具鲁棒性的大图顶层找出概略定位。
- 利用保存下的级联池化网络索引信息，系统指引着搜索路径回溯进入下层的高精度定位区域，最后在底部的基础 $40 \times 40$ 响应层产出经过全局鲁棒筛选后的极高精度候选对应点集。

### 1.5 步骤四：几何异常值剔除与单应性矩阵估计

- 剔除离群点：对候选对应点进行几何一致性检查（如结合 RANSAC 技术）全盘剔除所有空间错误对应的 outliers。
- 在干净平滑的内点集合上，最终估计并确定 homography（平面单应性矩阵）。
- 对后续执行：使用反向几何变换将热红外图像映射到可见光坐标系，并在浮点坐标插值（如双线性插值 warpPerspective 机制）时保持亚像素一致性，避免块状失真现象发生。

## 2. 工程契约对齐与约束边界

- 领域隔离：匹配层代码仅负责去畸变后的 RGB 与热红外影像之间的准确几何锚定计算，并输出对应结果，绝不允许直接执行三维辐射校正或融合赋值行为。
- 输出契约：工程中对应的匹配点结构须至少涵盖 `rgb_x`、`rgb_y`、`thermal_x`、`thermal_y`、`is_inlier`、`pyramid_level`、`patch_size`、`score` 及 `local_confidence`。
- 上下文追踪：总输出对象应一并记录 `confidence`、`match_quality`、`runtime_ms` 和算法的版本标识。如需附加金字塔中间态用于诊断，须一并绑定关联记录保存。具体依照 [docs/file_formats.md](docs/file_formats.md)。

## 3. 运行强制约束

- 绝对禁用违规算法主链：如前查验，生产链路及匹配组件包内严禁使能 SIFT、SURF、ORB、PC、RIFT 或 FSC。必须保证主调用逻辑落在实现上述四步的 `TWMM/TAMM_clean` 上。
- 不得对 TWMM 核心作非受控魔改：工程适配代码只做图像载入、金字塔参数等注入、超参数暴露管理并回收结果。搜索半径、原子块配置均应源自 [docs/runtime_config.md](docs/runtime_config.md) 且不可硬编码。
- TIFF 限定：强制链路接收去畸变版热红外 TIFF（而非低宽容度的 JPG 预览图）进入对应匹配体系。

## 4. 失败降级模式响应

如果面对：
- 热红外端过度模糊或纹理极弱；
- 飞行导致两重影像重叠率大幅低于阈值；
- 目标本身包含极大的景深视差以至于单平面单应性无效；
- 去畸变或基础像素对齐由于先验输入错误而尚未成功……

这最终会导致在匹配步骤四收到的内点集极少，产生极为畸变的单应映射。此时模块必须利用打分阈限输出显式的“低置信失败标识”，切断后继链路进行伪对应点绑定，安全记录该帧的匹配阻断并生成诊断日志。
